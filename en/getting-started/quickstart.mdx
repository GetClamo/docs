The `clamo` CLI orchestrates your local development environment. It manages services, databases, and dependencies.

## Prerequisites

| Software | Version | Purpose |
|----------|---------|---------|
| Node.js | 20+ | Frontend and TypeScript services |
| pnpm | 10+ | Package manager |
| Python | 3.10+ | CLI and Python services |
| uv | Latest | Python package manager |
| Docker | Latest | CEJ extractor and discovery workers |

### Install uv

<CodeGroup>
```bash Linux/macOS
curl -LsSf https://astral.sh/uv/install.sh | sh
```

```powershell Windows
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

```bash Homebrew
brew install uv
```
</CodeGroup>

Restart your terminal, then verify:

```bash
uv --version
```

## Installation

<Steps>
  <Step title="Clone the monorepo">
    ```bash
    git clone git@github.com:GetClamo/clamo.git
    cd clamo
    ```
  </Step>

  <Step title="Install the CLI">
    ```bash
    cd clamo-setup
    uv tool install . --force
    ```

    Restart your terminal, then verify:
    ```bash
    clamo --help
    ```
  </Step>

  <Step title="Run setup wizard">
    From the clamo root directory:
    ```bash
    cd /path/to/clamo
    clamo dev setup
    ```

    The wizard will ask you to select which services to run.
  </Step>

  <Step title="Start services">
    ```bash
    clamo dev start
    ```
  </Step>
</Steps>

<Note>
Always run `clamo` commands from the clamo root directory (where `clamo-web-app`, `clamo-cases`, etc. are located).
</Note>

## Commands

### Development

| Command | Description |
|---------|-------------|
| `clamo dev setup` | Interactive environment setup |
| `clamo dev start` | Start all configured services |
| `clamo dev start --rebuild` | Start and rebuild Docker containers |
| `clamo dev stop` | Stop Docker services |
| `clamo dev restart` | Restart all services |
| `clamo dev restart --rebuild` | Restart and rebuild containers |

### Database

| Command | Description |
|---------|-------------|
| `clamo db migrate stg` | Run migrations on staging |
| `clamo db migrate prd` | Run migrations on production |
| `clamo db status stg` | Check migration status |
| `clamo db studio stg` | Open Prisma Studio |

### CLI Management

| Command | Description |
|---------|-------------|
| `clamo update` | Update CLI from source |
| `clamo uninstall` | Uninstall CLI |

## Daily Workflow

```bash
# Initial setup (once)
clamo dev setup

# Start services
clamo dev start

# ... work on your code ...
# Ctrl+C to stop

# Restart if needed
clamo dev restart

# Rebuild containers if dependencies changed
clamo dev restart --rebuild
```

## Hot Reload

All services have hot reload enabled:

| Service | Hot Reload |
|---------|------------|
| clamo-web (Next.js) | Automatic |
| clamo-cases | Via `pnpm dev` |
| clamo-tenant | Via `pnpm dev` |
| luna (LangGraph) | Via `langgraph dev` |
| clamo-tools | Via `uvicorn --reload` |

## When to Rebuild Containers

**Rebuild needed:**
- Dependencies changed (`pyproject.toml`, `uv.lock`)
- Dockerfile changed
- Base image updated

```bash
clamo dev restart --rebuild
```

**No rebuild needed:**
- Python code changes (hot reload)
- `.env` changes
- Runtime configuration changes

## Configuration

Configuration is saved in `clamo.dev.config.json` in the root directory. Run `clamo dev setup` anytime to change your configuration.

## Troubleshooting

<AccordionGroup>
  <Accordion title="clamo: command not found">
    Restart your terminal after installing the CLI. If using development installation, activate the virtualenv:
    ```bash
    cd clamo-setup
    source .venv/bin/activate
    ```
  </Accordion>

  <Accordion title="Docker containers won't start">
    Verify Docker is running:
    ```bash
    docker ps
    ```
    If there are orphan containers:
    ```bash
    clamo dev stop
    docker system prune
    clamo dev start --rebuild
    ```
  </Accordion>

  <Accordion title="Port already in use">
    Check which process is using the port:
    ```bash
    lsof -i :3000  # web
    lsof -i :4000  # cases
    lsof -i :4001  # tenant
    ```
  </Accordion>
</AccordionGroup>
